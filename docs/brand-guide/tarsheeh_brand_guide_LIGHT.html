Scope

This document defines the light theme companion for Tarsheeh.cv. It covers only what changes between the dark and light modes: colour tokens, surface treatments, the logo on light, and UI components rendered on light.

Brand essence, mission, personality, voice, brand copy, phrases, microcopy, typography, type scale, spacing, and grid rules are unchanged. Refer to tarsheeh_brand_guide.html and tarsheeh_brand_guide_AR.html for those sections.

What stays the same

  • Cormorant Garamond for English display
  • DM Sans for English interface
  • Amiri for Arabic display
  • Cairo for Arabic interface
  • Type scale, weights, and tracking
  • Spacing tokens (xs through 2xl)
  • Logo wordmark, lockup, clear space rules
  • All copy, phrases, and microcopy strings
  • Border thicknesses and corner mark motif

What this document covers

  • Light theme colour tokens (8 values)
  • Dark to light token mapping table
  • Gold weighting on warm ivory surfaces
  • Logo treatment on light backgrounds
  • Buttons, badges, inputs on light
  • Candidate card and ranking on light
  • Live agent trace panel on light
  • RTL preview on light
  • Accessibility and contrast notes

03

Gold Weighting on Light

The relationship between gold tones inverts when moving from dark to light. On dark surfaces, Gold Light (#D4AE5A) carries the highlight while Gold (#B8922E) recedes into accent rules. On light surfaces, Gold remains the primary brand colour, but any gold-coloured text must use Gold Deep (#8C6E1F) to achieve readable contrast against the ivory page.

Use Gold (#B8922E) for

  • Decorative rules and 1px dividers
  • Corner mark motif on hero sections
  • Active border on focused inputs
  • Score bar fills and progress indicators
  • The .CV accent in the wordmark
  • Hover state borders

Use Gold Deep (#8C6E1F) for

  • Section number labels (01, 02, 03)
  • Numeric scores and values (e.g. 94)
  • Gold-coloured headings inside cards
  • Primary button background fill
  • Secondary button text colour
  • Any gold text smaller than 16px

Use Gold Light (#D4AE5A) for

  • Decorative use only on dark blocks
  • Dark-themed elements within light pages
  • Never as text on light backgrounds

Avoid on Light

  • Gold Light (#D4AE5A) as body or heading text
  • Pale gold (#EDD38A) anywhere except on dark surfaces
  • Pure black text (use Ink Primary instead)
  • Pure white surfaces (use Page Ivory instead)
  • Gold borders thicker than 1px outside CTAs

Accessibility & Contrast

Why Gold Deep replaces Gold for text

Gold (#B8922E) on Page Ivory (#F7F1E1) measures roughly 3.1 : 1 — below the WCAG AA threshold of 4.5:1 for body text. Gold Deep (#8C6E1F) measures 5.4 : 1, which clears AA for normal text and AAA for large text. Use Gold Deep wherever gold appears as readable text. Use Gold for purely decorative elements like rules, dividers, and focus borders, where contrast matters less than colour identity.

Body text contrast

Ink Primary (#1C180F) on Page Ivory (#F7F1E1) measures 14.2 : 1, well above AAA. The off-black is intentional. Pure #000000 on warm ivory creates a harsh visual seam that the warm near-black avoids, while still delivering excellent readability across screen and print.

Theme switching

When implementing the theme toggle, drive all colour values from CSS custom properties on :root[data-theme="light"] and :root[data-theme="dark"]. The token names in the mapping table (section 02) remain identical across both themes so component CSS does not need conditional logic. Only the values change.
